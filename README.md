# Курсовой терминал — USD/RUB · USDT/RUB

Статический сайт-монитор курсов по 12 источникам с дашбордом-карточками,
интерактивным графиком, фильтрами, аналитикой и поиском арбитража.

## Файлы
- `index.html` — вёрстка, стили и вся клиентская логика
- `config.js` — конфигурация: **хеш пароля** и параметры обновления
- `xlink-logo.png` — логотип на странице входа
- `netlify.toml` — конфиг Netlify (функции + маршрут `/api/rates`)
- `netlify/functions/` — серверные модули получения курсов (по модулю на источник)
- `README.md` — этот файл

## Запуск локально (без сервера)
Откройте `index.html` в браузере. Серверных функций не будет, поэтому сайт
автоматически переходит в **демо-симуляцию** (курс ЦБ РФ при этом всё равно
тянется по открытому API). Пароль по умолчанию: **012345678**.

## Локальная отладка с реальными данными
Нужен Netlify CLI (он поднимет и статику, и функции):
```bash
npm i -g netlify-cli
cd rate-monitor
netlify dev
```
Откроется адрес вида `http://localhost:8888`, где `/api/rates` уже работает.

## Деплой на Netlify
Перетащите папку **rate-monitor целиком** в Netlify (Add new site → Deploy manually)
либо подключите Git-репозиторий. Netlify сам увидит `netlify.toml`, соберёт
функции из `netlify/functions` и поднимет маршрут `/api/rates`. Build command —
пустой, publish directory — корень.

## Какие данные реальные, а какие «демо»
Реальные (сервер ходит к источникам, CORS там не мешает):
- **ЦБ РФ** — официальный курс USD/RUB (`cbr-xml-daily.ru`).
- **Binance** — P2P-курсы USDT/RUB (покупка/продажа).
- **Bybit** — P2P-курсы USDT/RUB (покупка/продажа).
- **Рыночный USD/RUB** — открытый FX-API (`open.er-api.com`) как ориентир.

Остальные площадки (Investing, TokenSpot, Alansary, Profinance, TradingView,
Coinbase, Crypto.com, Gate, MEX) помечены бейджем **«демо»**: у них нет
бесплатного публичного API, поэтому их значения привязаны к реальным якорям
(P2P-среднее и официальный курс) и слегка колеблются. Это честная заглушка,
а не реальные котировки этих брендов.

> Важно про P2P-эндпоинты Binance/Bybit: они **неофициальные** (внутренние URL
> p2p-разделов). Обычно работают, но биржа их не гарантирует — формат может
> измениться, а запросы из некоторых дата-центров иногда гео-блокируются.
> Если такой источник недоступен, его карточка покажет статус «ошибка», а
> остальной сайт продолжит работать. Логика изолирована в
> `netlify/functions/lib/binanceP2P.js` и `bybitP2P.js`.

> Если у Bybit покупка и продажа окажутся перепутаны — поменяйте местами
> аргументы в `getBybit()` (файл `lib/bybitP2P.js`): `side("1")` и `side("0")`.

## Подключение Twelve Data (рыночные USD/RUB и USDT/RUB)
Рыночные курсы тянутся из Twelve Data, если задан ключ. Без ключа сайт
автоматически использует бесплатный резерв (FX + CoinGecko) — тоже реальные данные.

1. Зарегистрируйтесь на twelvedata.com, скопируйте **API key** в личном кабинете.
2. В Netlify: **Site configuration → Environment variables → Add a variable**.
   Имя: `TWELVEDATA_KEY`, значение — ваш ключ. (Ключ в код не вписывается и в браузере не виден.)
3. **Trigger deploy** (Deploys → Trigger deploy), чтобы переменная применилась.
4. Проверьте `/api/rates` → в блоке `market` поле `source` должно стать `"twelvedata"`.
   Если там `"free-fallback"` — ключ не подхватился или закончились кредиты.

**Про лимиты бесплатного тарифа.** Free ≈ 800 кредитов/день, один запрос USD/RUB+USDT/RUB
стоит ~2 кредита. Для теста хватает. Для круглосуточной работы увеличьте кэш: в
`netlify/functions/rates.js` поменяйте `CACHE_MS = 10000` на `60000` и больше — так
один ключ обслужит любой трафик. Когда кредиты кончатся, сайт сам перейдёт на
бесплатный резерв и не сломается.


Создайте новый модуль в `netlify/functions/lib/<имя>.js` по образцу существующих
(экспортирует функцию, возвращающую `{buy, sell}` или одно число), затем
подключите его в `netlify/functions/rates.js` в блоке `Promise.allSettled` и
запишите результат в `sources.<id>` с `real:true`. Демо-запись для этого id из
массива `DEMO` уберите.

## Смена пароля
Пароль хранится SHA-256 хешем в `config.js`. Новый хеш:
```bash
printf 'НОВЫЙ_ПАРОЛЬ' | sha256sum
```

> Важно: это статический сайт без сервера — проверка пароля целиком
> выполняется в браузере, поэтому хеш всё равно виден в исходниках.
> Хеширование убирает пароль открытым текстом, но для настоящей защиты
> проверку нужно выносить на бэкенд (например, Netlify Functions) и
> держать секрет в переменной окружения.

## Поведение сессии
- Интерфейс — светлая тема.
- Настройки (режим покупка/продажа, валютная пара, фильтры, источники на графике,
  порядок карточек, цвета линий) сохраняются в `localStorage` и переживают
  обновление страницы.
- Авторизация в памяти не сохраняется: при обновлении/закрытии/повторном открытии
  вкладки пользователь разлогинивается и снова видит страницу входа — а настройки
  восстанавливаются после нового входа.
